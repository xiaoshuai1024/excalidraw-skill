# 软件研发体系所有图类型及其正规画法

本文件是 skill 的**权威引用**。当用户要求画图时，skill 根据本文件确定最正规的画法。除非用户明确要求简化，否则**必须使用完整的正规画法**。

## 一、UML 结构图

### 1. 类图（Class Diagram）
- **类**：三层矩形框（类名/属性/方法）。属性前 `+`public `-`private `#`protected。
- **关系**：
  - 继承：空心三角实线（子→父）
  - 实现：空心三角虚线（实现→接口）
  - 关联：实线
  - 聚合：空心菱形实线（整体◇→部分）
  - 组合：实心菱形实线（整体◆→部分）
  - 依赖：虚线开口箭头（依赖方→被依赖方）
- **多重性**：`1` `0..1` `*` `1..*`，标注在线末端
- **接口**：`<<interface>>` stereotype 或圆圈

### 2. 组件图（Component Diagram）—— 当前有但需升级
- **组件**：矩形 + `<<component>>` stereotype，或右上角带组件图标
- **接口**：⊙（lollipop = provided），半圆弧线（socket = required）
- **端口**：组件边上的小方块
- **依赖**：虚线开口箭头
- **委托连接器**：连接外部接口到内部组件
- **正则画法**：组件名，stereotype，provided/required 接口，内部子结构

### 3. 部署图（Deployment Diagram）
- **节点**：立方体（3D 矩形），标注硬件/OS/运行时
- **工件**：`<<artifact>>` 矩形，放在节点内
- **通信路径**：节点间实线，标注协议（HTTP/TCP/gRPC）
- **设备**：`<<device>>` stereotype
- **执行环境**：`<<executionEnvironment>>`（如 Docker、JVM）

### 4. 包图（Package Diagram）
- **包**：文件夹图标（大矩形+左上角小矩形）
- **依赖**：虚线开口箭头
- **包含**：带加号的嵌套

### 5. 组合结构图
- 展示类/组件的内部结构
- 部件（part）+ 连接器（connector）+ 端口

### 6. 对象图
- 类图的实例化版本。对象名下方加下划线。`对象名 : 类名`

## 二、UML 行为图

### 7. 用例图（Use Case Diagram）
- **参与者**：火柴人图标
- **用例**：椭圆，内含动宾短语（如"下单"）
- **边界**：矩形框包围用例，顶部标注系统名
- **关系**：
  - 关联：实线连接 actor 和 use case
  - 包含 `<<include>>`：虚线+开口箭头（必须执行）
  - 扩展 `<<extend>>`：虚线+开口箭头（条件执行）
  - 泛化：空心三角实线（actor 或 use case 的继承）

### 8. 活动图（Activity Diagram）
- **初始/终止**：实心圆 / 实心圆+外环
- **动作**：圆角矩形
- **决策/合并**：菱形
- **分叉/汇合**：粗横线（并发）
- **泳道**：垂直/水平分区
- **对象流**：虚线开口箭头
- **信号**：发送（凸五边形）、接收（凹五边形）

### 9. 状态机图（State Machine Diagram）
- **状态**：圆角矩形，可分格（名称格/内部活动格）
- **初始**：实心圆；**终止**：空心圆带实心点
- **转换**：实线+开口箭头，标签 `事件[条件]/动作`
- **内部活动**：`entry/动作` `do/活动` `exit/动作`
- **复合状态**：大圆角矩形包含子状态
- **历史状态**：圆圈带 H

### 10. 时序图（Sequence Diagram）—— 当前有但需升级
- **参与者**：顶部矩形框（actor 用火柴人）
- **生命线**：**虚线**从参与者向下
- **激活框**：细长矩形覆盖在生命线上，表示该对象正在执行
- **消息**：
  - 同步消息：实线+实心箭头
  - 异步消息：实线+开口箭头
  - 返回消息：虚线+开口箭头（非必须，重要时才标）
  - 自调用：回到同一条生命线的箭头
- **创建/销毁**：`<<create>>` / 大 X
- **片段**（组合片段）：
  - `alt` 选择、`opt` 可选、`loop` 循环、`par` 并行
  - `ref` 引用其他图
- **正规画法**必须包含：Gates（边界）、激活框、消息编号、关键返回消息

### 11. 通信图（Communication Diagram）
- 类似时序图但强调对象间关系。对象用矩形，连接用实线，消息标在线上带编号。

### 12. 交互概览图
- 活动图 + 时序图的混合。节点是小的时序图引用。

### 13. 时间图
- 横轴时间，纵轴状态，用于实时/嵌入式系统。

## 三、业务/流程建模

### 14. 流程图（Flowchart）—— 当前基本正确
- 开始/结束：椭圆
- 处理：矩形
- 判断：菱形
- 流向：箭头
- **泳道版**：加上垂直/水平泳道

### 15. BPMN（Business Process Model and Notation）
- **池**：大矩形框（一个组织）
- **泳道**：池内的水平分区（角色）
- **事件**：圆圈（开始=细线、中间=双线、结束=粗线）
- **活动**：圆角矩形
- **网关**：菱形（排他=X、并行=+、包容=〇）
- **消息流**：虚线+空心箭头（跨池）
- **顺序流**：实线+实心箭头（池内）
- **数据对象**：右上角折角的矩形

### 16. 数据流图（DFD）
- **外部实体**：矩形（通常双线边框）
- **处理**：圆角矩形或圆形
- **数据存储**：右边开口的矩形
- **数据流**：箭头+标签

## 四、架构/系统设计

### 17. 业务架构图 —— 当前有
- 分层模型（业务能力层/流程层/数据层）
- 能力热力图（红黄绿）

### 18. C4 Model（软件架构专用）
- **Level 1 系统上下文**：系统 + 用户 + 外部系统，实线+虚线
- **Level 2 容器图**：应用/数据库/文件系统，带技术标注（如 Spring Boot, PostgreSQL）
- **Level 3 组件图**：每个容器内部的组件
- **Level 4 代码图**：类/接口

### 19. 部署架构图 —— 当前有
- 节点+通信路径+协议标注+安全区域

---

## 五、数据/数据库

### 20. ER 图（Crow's Foot）—— 当前需升级
- **实体**：矩形
- **属性**：PK 标 `*` 或下划线，FK 标 `FK`，放在实体矩形内
- **关系线**：实体间的连线
- **基数标记**（在线的两端）：
  - `|` = 恰好一（mandatory one）
  - `○` = 零（optional）—— 内圈
  - `|<` = 一到多（mandatory many，|<是三叉的crow's foot）
  - `○<` = 零到多（optional many）
- **关系动词**：线上标注
- **关联实体**：M:N 关系必须拆分
- **正规画法**：必须有完整的属性列表（PK/FK标注）+ Crow's Foot 基数 + 关系动词

### 21. 实体生命期图
- 横轴时间，纵轴状态，展示实体在各阶段的状态与操作

## 六、测试/质量

### 22. FMEA / 故障树
- 故障（矩形）+ 逻辑门（AND/OR）+ 基本事件（圆形）

### 23. 因果图（鱼骨图 / Ishikawa）
- 水平主干（问题在箭头右侧）+ 斜线分支（原因类别）+ 子分支

## 七、研发管理

### 24. 甘特图 / Roadmap
- 横轴时间，纵轴任务，横条表示工期，菱形表示里程碑

### 25. 燃尽图
- 横轴时间，纵轴工作量，理想线 vs 实际线

### 26. 组织架构图
- 矩形框+连线。层级结构。

---

## 八、用户/产品

### 27. 用户旅程图（User Journey Map）—— 当前需升级
- **阶段**：5-7 个（认知→考虑→注册→首次使用→日常→留存→推荐）
- **每阶段的行**：
  1. 触点（Touchpoints）
  2. 用户动作（Actions）
  3. 用户想法（Thoughts）
  4. 情绪曲线（Emotion）：-2 到 +2 的线图
  5. 痛点（Pain Points）
  6. 机会（Opportunities）
- **Persona**：左侧标注

### 28. 服务蓝图（Service Blueprint）
- 类似旅程图，但多了前台行为/后台行为/支持过程三层

### 29. 思维导图 —— 当前基本正确
- 中心节点+放射分支+层级

### 30. 用户故事地图
- 横轴：用户活动；纵轴：故事优先级（critical→nice to have）

### 31. 同理心地图
- 四象限：说/做/想/感觉

## 九、专用图

### 32. 网络拓扑图 —— 当前有
- 设备符号+区域分组+连接线+协议标注

### 33. CI/CD 流水线
- 阶段矩形（build/test/deploy）+箭头+工具标注

### 34. Wardley Map
- 纵轴（价值链 visibility），横轴（成熟度 evolution），组件气泡+连线

### 35. 决策树
- 根节点（矩形）→ 分支（线）→ 叶子（标签）。每层一个属性。

### 36. 状态转移矩阵
- 表格形式，行列都是状态，单元格是触发事件

## 十、研发协作与管理

### 37. RACI 矩阵
- 表格：行=任务/决策，列=角色/人员
- 每个格子填：R(执行者)、A(负责人)、C(咨询)、I(知会)
- 确保每个任务有且仅有一个 A

### 38. 影响-努力矩阵（Impact-Effort Matrix）
- 四象限：横轴=努力度(低→高)，纵轴=影响力(低→高)
- 右上=优先做，左上=速赢，右下=不值得，左下=可考虑
- 气泡大小=价值/成本

### 39. 风险矩阵（Risk Matrix）
- 横轴=可能性，纵轴=影响度
- 红黄绿色块区分风险等级
- 气泡表示具体风险项

### 40. 干系人地图（Stakeholder Map）
- 横轴=兴趣度/影响力，纵轴=权力
- 四象限：高权高趣=重点管理，高权低趣=保持满意，低权高趣=保持知情，低权低趣=监控
- 气泡代表具体干系人

### 41. 价值流图（Value Stream Map）
- 横轴=流程步骤，标注每个步骤的 LT(前置时间) / PT(处理时间)
- 上下=信息流 vs 物料流
- 底部汇总：增值时间 vs 非增值时间

### 42. 看板（Kanban Board）
- 列=状态（Todo→Doing→Review→Done）
- 卡片=工作项，标注负责人、优先级、阻塞标记
- WIP 限制标注在列顶部

### 43. 燃起图（Burnup Chart）
- 横轴=时间/迭代，纵轴=工作量
- 两条线：范围线(总需求，会上移) vs 完成线(已交付)

## 十一、领域驱动设计（DDD）

### 44. Event Storming 事件风暴
- **领域事件**：橙色便利贴（过去式动词）
- **命令**：蓝色便利贴（动词短语）
- **聚合**：黄色便利贴
- **外部系统**：粉色便利贴
- **策略/政策**：紫色便利贴（始终/条件）
- 自左向右时间线排列

### 45. Context Map（限界上下文映射）
- 每个限界上下文=一个矩形
- 上下文间关系线标注：
  - U=上游/D=下游（Upstream/Downstream）
  - ACL=防腐层（Anti-Corruption Layer）
  - OHS=开放主机服务（Open Host Service）
  - PL=发布语言（Published Language）
  - CF=遵奉者（Conformist）
  - SK=共享内核（Shared Kernel）

### 46. 领域架构图
- 核心域/支撑域/通用域三个同心圆
- 内部标注子域和聚合

### 47. 聚合设计图
- 根实体（聚合根）=带星号的矩形
- 子实体=矩形，值对象=圆角矩形
- 边界用虚线框标注聚合范围

## 十二、DevOps / SRE

### 48. Git 分支策略图
- **Git Flow**：main → develop → feature/release/hotfix 分支
- **GitHub Flow**：main + feature branches + PR
- **Trunk-Based**：main + short-lived feature branches
- 用分支线+合并箭头表示

### 49. 告警升级流程（Alert Escalation）
- 层级：L1值班→L2专家→L3架构师→管理层
- 每个层级标注响应时间（5min/15min/30min/1h）
- 升级条件标注

### 50. 系统景观图（System Landscape）
- 所有系统/服务=矩形框，按业务域分组
- 集成关系=线条，标注协议和数据流向
- 系统状态标注（RUNNING/DEGRADED/DOWN）

### 51. 技术雷达（Technology Radar）
- 四个同心圆环：采纳/试验/评估/暂缓（Adopt/Trial/Assess/Hold）
- 四个扇形区域：技术/平台/工具/语言框架
- 每个技术项=圆点放入对应位置

### 52. 4+1 架构视图
- **逻辑视图**：类图/包图
- **开发视图**：模块/子系统分解
- **进程视图**：进程/线程/通信
- **物理视图**：部署图
- **场景视图**：用例图（+1）

### 53. NFR / 质量属性树
- 根节点=系统质量
- 一级分支：性能/安全/可用性/可维护性/可测试性
- 每个属性下拉出具体指标（如性能→响应时间<200ms、吞吐量>1000TPS）

### 54. 容量规划图
- 横轴=时间（月/季度），纵轴=资源（CPU/内存/存储）
- 历史线+预测线+扩容节点

---

## 十三、API / 集成

### 55. API 流转图（Service Interaction）
- 服务=矩形框，标注服务名和技术栈
- 调用链=箭头，标注协议（gRPC/HTTP/MQ）+ 数据结构
- 同步=实线，异步=虚线

### 56. 消息流转图（Message Flow）
- Topic/Queue=梯形或菱形
- 生产者→Topic→消费者
- 标注消息格式、QPS、延迟

### 57. OpenAPI / 接口全景图
- 资源路径树
- HTTP 方法+响应码

---

## 十四、产品与战略

### 58. Kano 模型（Kano Model）
- 横轴=功能实现度，纵轴=用户满意度
- 三条曲线：基本需求(底线)、期望需求(线性)、兴奋需求(上翘)
- 标注当前产品位置

### 59. 产品树（Feature Tree / Product Tree）
- 树干=产品名
- 主干分支=功能域，子分支=具体功能
- 叶子颜色表示状态（已上线/开发中/计划中）

### 60. 商业模式画布（Business Model Canvas）
- 九宫格：客户细分/价值主张/渠道/客户关系/收入流/核心资源/关键活动/关键伙伴/成本结构

### 61. 用户画像卡（Persona Card）
- 矩形框内：照片占位、姓名、年龄、职业、目标、痛点、行为模式

### 62. 竞品分析矩阵
- 横轴=某维度，纵轴=另一维度
- 各竞品气泡标注位置

---

## 十五、安全

### 63. 威胁建模图（STRIDE / Threat Model）
- 数据流图基础上标注威胁
- S=欺骗、T=篡改、R=抵赖、I=信息泄露、D=拒绝服务、E=提权
- 每类威胁用不同颜色标记

### 64. 攻击面图（Attack Surface）
- 系统边界+暴露的端点/端口/API
- 红色标注高风险暴露面

---

## 使用原则

1. **默认用最正规画法**——完整 notation、标注、图例
2. **简化版只在用户明确要求时使用**
3. **图例是必选项**——每种图必须带 legend
4. **选图决策**：用户描述 → 匹配最合适的图类型 → 用正规画法
5. **总数 64 种**：UML(13) + 业务流程(3) + 架构(5) + 数据(2) + 测试质量(2) + 管理(8) + 用户产品(5) + 专用(5) + 研发协作(7) + DDD(4) + DevOps/SRE(7) + API集成(3) + 产品战略(5) + 安全(2)
